# Available Rules

The --fix option on the command line (opens new window) automatically fixes problems reported by rules which have a wrench 🔧 below.
The rules with the following star ⭐️ are included in the plugin:@ota-meshi/svelte/recommended config.

Rule ID Description
@ota-meshi/svelte/button-has-type disallow usage of button without an explicit type attribute
@ota-meshi/svelte/comment-directive support comment-directives in HTML template ⭐️
@ota-meshi/svelte/no-at-debug-tags disallow the use of {@debug} ⭐️
@ota-meshi/svelte/no-at-html-tags disallow use of {@html} to prevent XSS attack ⭐️
@ota-meshi/svelte/no-dupe-else-if-blocks disallow duplicate conditions in {#if} / {:else if} chains ⭐️
@ota-meshi/svelte/no-inner-declarations disallow variable or function declarations in nested blocks ⭐️
@ota-meshi/svelte/no-target-blank disallow target="_blank" attribute without rel="noopener noreferrer"
@ota-meshi/svelte/no-useless-mustaches disallow unnecessary mustache interpolations 🔧
@ota-meshi/svelte/prefer-class-directive require class directives instead of ternary expressions 🔧
@ota-meshi/svelte/spaced-html-comment enforce consistent spacing after the <!-- and before the --> in a HTML comment 🔧
@ota-meshi/svelte/system system rule for working this plugin ⭐️
